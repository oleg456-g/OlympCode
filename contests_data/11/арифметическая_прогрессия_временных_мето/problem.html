В старом городе на башне ратуши установлены необычные механические часы. Каждый день хранитель записывает все моменты, когда часы били: иногда механизм работал исправно, а иногда от старости мог дать сбой и пробить не в тот момент.

Известно, что правильный бой часов должен происходить ровно через одинаковый интервал времени. Хранитель хочет найти самый длинный фрагмент журнала, который мог быть получен от исправно работающего механизма.

Фрагмент записей считается правильной цепочкой, если:

  • времена в нём идут строго по возрастанию;
  • разность между соседними моментами одинакова для всех соседних пар.

Вам дан журнал из N различных моментов времени за один день. Нужно определить:

  1. максимальное количество записей, которые можно выбрать из журнала так, чтобы они образовали правильную цепочку;
  2. сами эти моменты времени.

Если существует несколько правильных цепочек максимальной длины, нужно выбрать ту, у которой первый момент времени меньше. Если и таких несколько, выбрать ту, у которой второй момент времени меньше. Если правильная цепочка состоит из одного момента, нужно вывести только его.

Формат ввода

На первой строке записано целое число N — количество записей в журнале (1 ≤ N ≤ 2000). Записи не обязательно в порядке возрастания (может быть случайный порядок).

На следующих N строках записаны моменты времени в формате HH:MM:SS. Все моменты различны.

Формат вывода

На первой строке выведите максимальное количество записей в правильной цепочке.

Далее выведите моменты этой цепочки по порядку (в возрастающем порядке), каждый на новой строке, в формате HH:MM:SS.
