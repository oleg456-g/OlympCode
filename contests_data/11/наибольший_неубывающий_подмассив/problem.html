На далёком северном берегу стоит старый маяк. Каждую ночь смотритель записывает в журнал силу свечения фонаря по минутам. Если ночью был туман, то некоторые минуты получались особенно важными: именно в такие моменты подряд идущие значения яркости образовывали «устойчивый луч».

Смотритель называет отрезок устойчивым, если на нём яркость ни разу не уменьшалась, то есть каждое следующее число не меньше предыдущего.

В архиве маяка нашли список яркости фонаря за N минут подряд. Теперь нужно определить, какой самый длинный устойчивый отрезок можно найти в этом списке.

Если таких отрезков несколько, нужно вывести тот, который начинается раньше.

Требуется написать программу, которая по журналу яркости определит:

  1. длину самого длинного устойчивого отрезка;
  2. номер минуты, с которой он начинается;
  3. сам этот отрезок.

Если в журнале только одна запись, то она сама считается устойчивым отрезком длины 1.

Формат ввода

На первой строке задано целое число N — количество записей в журнале (1 ≤ N ≤ 100000).

На второй строке через пробел записаны N целых чисел a1, a2, ..., aN — яркость фонаря по минутам (0 ≤ ai ≤ 109).

Нумерация минут начинается с 1.

Формат вывода

На первой строке выведите длину самого длинного устойчивого отрезка.

На второй строке выведите номер минуты, с которой начинается этот отрезок.

На третьей строке выведите элементы этого отрезка через пробел.
