# OlympCode

Веб-платформа для проведения олимпиад по программированию — автоматическая проверка решений в изолированной песочнице, поддержка форматов задач Polygon и МОШ, гибкая система оценивания (ICPC / IOI). Работает на проде: [olympcode.ru](https://olympcode.ru/contests/)

## О проекте

OlympCode — это онлайн-джадж (судейская система): школьник отправляет код решения через браузер, система компилирует его, прогоняет на наборе тестов в изолированном окружении, проверяет вывод чекером и выставляет вердикт и баллы.

Проект вырос из реальной потребности: нужных олимпиадных задач не было загружено на Codeforces, а готовить школьников к олимпиадам на чём-то было нужно. Поэтому появилась своя платформа, куда можно загрузить любую задачу из стандартных форматов (Polygon, МОШ) и сразу проводить на ней тренировки и соревнования. Сейчас система используется для реальных олимпиад и подготовки школьников.

## Возможности

- **Автоматическая проверка решений** на C++ и Python с компиляцией и прогоном по тестам
- **Изоляция выполнения** через [isolate](https://github.com/ioi/isolate) — решения участников запускаются в песочнице с лимитами по времени, памяти и числу процессов; вредоносный код не имеет доступа к системе
- **Интерактивные задачи** — двусторонний диалог решения с интерактором в реальном времени
- **Форматы задач**: импорт архивов из [Polygon](https://polygon.codeforces.com/) (problem.xml, тесты, чекеры, группы) и формата МОШ (Московской олимпиады школьников)
- **Системы оценивания**:
  - ICPC — полный проход или ничего
  - IOI (подзадачи) — баллы по группам тестов с зависимостями между подзадачами и пропуском недостижимых групп
  - Весовые чекеры (testlib `quitp`) — частичные баллы за тест
- **Чекеры testlib** — компиляция и запуск авторских чекеров и интеракторов
- **Очередь тестирования** — параллельные воркеры, устойчивость к перезапуску (зависшие посылки восстанавливаются)
- **Админ-панель** — создание олимпиад, загрузка и редактирование задач, просмотр посылок, управление видимостью контестов
- **Прогресс тестирования в реальном времени** — участник видит на каком тесте сейчас идёт проверка (SSE)

## Технологии

| Слой | Технологии |
|------|-----------|
| Backend | Python, FastAPI, SQLAlchemy |
| База данных | SQLite (WAL-режим) |
| Frontend | Jinja2, CodeMirror, ванильный JS |
| Песочница | isolate (Linux namespaces, cgroups) |
| Компиляция | g++ (C++20/23), CPython |
| Инфраструктура | nginx, uvicorn, systemd, certbot (HTTPS) |
| Проверка | testlib (чекеры и интеракторы) |

## Архитектура

```
Браузер ──HTTPS──> nginx ──> uvicorn (FastAPI)
                                   │
                    ┌──────────────┼──────────────┐
                    │              │              │
              Роутеры/API    Импортёры       Воркер очереди
              (auth, tasks,  (Polygon, МОШ)  тестирования
               submissions,                       │
               admin)                       ┌──────┴──────┐
                    │                       │             │
                SQLite                  Песочница      Чекеры
                                        (isolate)     (testlib)
```

Поток проверки решения:
1. Участник отправляет код → посылка ставится в очередь (`PENDING`)
2. Воркер берёт посылку, компилирует решение
3. Каждый тест прогоняется в isolate-боксе с лимитами TL/ML
4. Вывод решения проверяется чекером, начисляются баллы
5. Для IOI-задач — подсчёт по подзадачам с учётом зависимостей групп
6. Вердикт и детализация по тестам сохраняются и стримятся участнику

## Безопасность

- Выполнение кода участников в изолированной песочнице (isolate) с жёсткими лимитами
- CSRF-защита (SameSite-куки + токены)
- Rate-limiting на авторизацию и отправку решений
- Защита от path traversal при доступе к файлам задач
- Хеширование паролей (bcrypt)
- Security-заголовки (X-Frame-Options, nosniff, Referrer-Policy)
- HTTPS через Let's Encrypt

## Установка и запуск

### Требования

- Linux (для песочницы isolate; разработка возможна на любой ОС, но проверка решений — только Linux)
- Python 3.11+
- g++ с поддержкой C++20/23
- [isolate](https://github.com/ioi/isolate) — песочница для безопасного запуска решений
- nginx (для прода)

### Локальный запуск

```bash
# 1. Клонировать репозиторий
git clone https://github.com/oleg456-g/OlympCode.git
cd OlympCode

# 2. Виртуальное окружение и зависимости
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 3. Настроить переменные окружения
cp .env.example .env
# В .env задать SESSION_KEY (любая длинная случайная строка)

# 4. Запустить
cd web
uvicorn main:app --reload
```

Откроется на `http://127.0.0.1:8000`. База данных SQLite создаётся автоматически при первом запуске.

### Настройка песочницы isolate

Для проверки решений нужен установленный и настроенный isolate:

```bash
# Установка
sudo apt install isolate

# Настройка user-namespaces для изоляции
# (в /etc/subuid и /etc/subgid добавить маппинг для пользователя isolate)

# testlib.h для компиляции чекеров и интеракторов
sudo cp testlib.h /usr/local/include/
```

Подробности настройки cgroups и namespaces — в документации isolate. Без настроенного isolate система работает в fallback-режиме (проверка без изоляции — только для разработки).

### Создание администратора

Первый пользователь с правами админа задаётся напрямую в БД (поле `is_admin`). После этого через админ-панель можно создавать олимпиады и загружать задачи.

## Развёртывание на проде

Прод работает на Linux-сервере:

- **nginx** проксирует запросы на uvicorn, отдаёт статику и файлы задач, ограничивает размер загрузок
- **uvicorn** запускает FastAPI-приложение на локальном порту
- **systemd** управляет процессом (автозапуск, перезапуск при падении, сброс зависших посылок при старте)
- **isolate** настроен с user-namespaces и cgroups для изоляции выполнения
- **certbot** (Let's Encrypt) — HTTPS с автообновлением сертификатов

Конфигурация безопасности на проде: HTTPS-only куки, firewall (UFW), SSH по ключу на нестандартном порту, ограничение прав процесса через systemd (`ReadWritePaths`, `NoNewPrivileges`).
